üéØ Understanding Computational Complexity

Not all languages are created equal. Some can be decided quickly, others slowly, and some cannot be decided at all! This hierarchy tells us fundamental truths about the limits of computation.

üîß The Famous Halting Problem

The Halting Problem asks: "Given a DTM M and input w, does M halt on w?"

Why is it undecidable?

Proof by Contradiction:
  1. Assume H exists: H(‚ü®M‚ü©, w) = "yes" if M halts on w, "no" otherwise
  2. Construct D: D(‚ü®M‚ü©) = if H(‚ü®M‚ü©, ‚ü®M‚ü©) = "yes" then loop forever, else halt
  3. What does D(‚ü®D‚ü©) do?
  4. If D halts on ‚ü®D‚ü©, then H(‚ü®D‚ü©, ‚ü®D‚ü©) = "yes", so D loops on ‚ü®D‚ü© ‚ö°
  5. If D loops on ‚ü®D‚ü©, then H(‚ü®D‚ü©, ‚ü®D‚ü©) = "no", so D halts on ‚ü®D‚ü© ‚ö°
  6. Contradiction! H cannot exist.
